What is a PCAP File?

PCAP (short for Packet Capture) is the standard file format used to store network packet data captured from live network interfaces. It was originally created for the libpcap library, which underlies both Wireshark and tcpdump.

A PCAP file is not a high-level text log. Instead, it is a sequential log of raw binary frames exactly as they were pulled from the network card, wrapped in lightweight headers that supply timestamps and size metadata.

tcpdump Command Reference

CLI / Raw capture

1. Device & Interface Selection

tcpdump -D
List all network interfaces available for capture.
tcpdump -i any
Capture packets on all available interfaces (good for general sweeps).

2. File Input / Output Operations

tcpdump -w capture.pcap
Write captured packets raw into 'capture.pcap' instead of printing.
tcpdump -r capture.pcap
Read and display packets from an existing PCAP file.

3. Filter Modifiers (BPF Syntax)

tcpdump port 80
Filter traffic where source or destination port is 80 (HTTP).
tcpdump host 192.168.1.1
Filter traffic with matching source or destination IP.
tcpdump src net 10.0.0.0/24
Filter traffic originating from a specific subnet range.

4. Tuning Output Output Details

tcpdump -n -nn
Don't resolve hostnames (-n) or ports (-nn) to text. Runs faster.
tcpdump -vvv -X
Maximum verbosity (-vvv) and show data payload in both Hex and ASCII (-X).

Wireshark Display Filters

GUI / Analysis

1. Filtering by Protocol

dns
Matches all DNS query and response packets.
http
Matches HTTP request/response packets (excludes HTTPS).

2. Filtering by Address (IP/MAC)

ip.addr == 192.168.1.1
Matches packets where either IP source or destination matches.
ip.src != 10.0.0.5
Matches packets where source IP is not the specified host.

3. Filtering by Port Numbers

tcp.port == 443
Matches packets using TCP source or destination port 443 (HTTPS).
udp.port == 53
Matches UDP DNS standard packets.

4. Advanced Payload & Logic Matching

http.request.method == "POST"
Matches HTTP packets where request method is POST.
tcp.flags.syn == 1 && tcp.flags.ack == 0
Matches only initial TCP SYN connection requests.
